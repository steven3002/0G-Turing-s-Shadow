
# 0G-Turing's-Shadow: MVP Backend Documentation

## 1. Project Overview
0G-Turing's-Shadow is a real-time, deterministic, high-concurrency spatial game engine built in Go. It is designed specifically to host autonomous AI agents in a social deduction environment (similar to Among Us). 

The backend architecture prioritizes strict thread safety, event-driven sensory broadcasting, and Sybil-resistant WebSocket connections.

## 2. Implemented Architecture & Features

### A. The State Access Layer (`internal/state`)
The core memory of the game. It is designed with strict encapsulation to prevent race conditions.
* **Granular Mutexes:** Global game state, individual rooms, and individual players all have their own `sync.RWMutex`.
* **Private Maps:** Underlying data structures (like room attendance or player lists) are unexported. They can only be mutated via strictly controlled methods (e.g., `room.AddToAttendance()`).
* **Zero Raw Exposure:** The engine interacts with the state via safe pointers and snapshot generation to prevent deadlocks.

### B. The Network Layer (`internal/network`)
A non-blocking, hub-and-spoke WebSocket architecture using `gorilla/websocket`.
* **Concurrent Pumps:** Every connected client gets its own dedicated `readPump` and `writePump` goroutine, ensuring a slow client never blocks the server.
* **The AI Gatekeeper:** A strict 150ms cryptographic challenge-response handshake enforced at the socket level. Human tools (like Postman) or slow scripts will timeout and be violently disconnected before they can touch the game state.

### C. The Core Engine & Event Loop (`internal/engine`)
The deterministic heartbeat of the game.
* **100ms Tick Loop:** A background worker that fires 10 times a second to evaluate timers and garbage collect fast-decaying spatial logs.
* **Matchmaker:** Automatically evaluates lobby size and securely initializes the match when 9 verified agents connect.
* **Action Handlers:** Validates and executes `MOVE`, `KILL`, `START_TASK`, `PANIC_BUTTON`, and `REPORT_BODY` commands, enforcing strict cooldowns (e.g., 1.5s migration lock, 5s kill cooldown).
* **Sensory Broadcaster (Context Injector):** Dynamically constructs JSON prompts (`PROMPT_05_STATE_TICK_UPDATE`, `PROMPT_06_LOCAL_MAP_UPDATE`) and pushes them exclusively to agents within line-of-sight.
* **Task Blindness:** The engine physically drops sensory payloads for any agent currently locked in a 3.0s `DOING_TASK` animation.

### D. The Social Deduction Engine (Meeting Phase)
A dedicated state machine that suspends spatial gameplay to force NLP negotiation.
* **Strict Timers:** Manages the 20s Chat Phase and 10s Voting Phase using absolute Unix timestamps to prevent network latency exploits.
* **Chat Rate Limiting:** Enforces a 0.6s cooldown between chat messages to prevent context window flooding.
* **Vote Resolution:** Automatically tallies votes, triggers ejection, reveals the Impostor status, evaluates win conditions, and seamlessly resets the board to the Cafeteria if the game continues.

## 3. Replication Guide (Local Testing)

To run the full end-to-end local simulation of the server and 9 AI agents, follow these steps.

### Prerequisites
* Go installed (`v1.20+` recommended)
* A bash-compatible terminal (Linux, macOS, or WSL on Windows)

### Step 1: Start the Server
Open a terminal instance, navigate to the root of the backend directory, and start the game engine.
```bash
go run cmd/server/main.go
```
*Expected Output:*
> Booting 0G-Turing-s-Shadow Backend...
> Starting Global Event Loop (Tick Rate: 100ms)...
> Server successfully listening on :8080

### Step 2: Spawn the AI Test Clients
Open a **second** terminal instance. To trigger the matchmaker, we need 9 verified WebSocket connections. We will use a bash loop to spawn 9 instances of our test client simultaneously in the background.
```bash
for i in {1..9}; do go run cmd/testclient/main.go & done
```

### Step 3: Observe the Game Loop Initialization
If successful, you will observe the following sequence across your terminals:

1.  **Handshake:** The server logs 9 rapid `Handshake successful. Agent authenticated as: agent_XXXX` messages.
2.  **Matchmaker:** The server counts the lobby `[MATCHMAKER] Lobby at X/9 players.` until it hits 9, then logs `Initializing Match...`
3.  **Role Assignment:** The test client terminal will explode with JSON as all 9 dummy clients receive their `PROMPT_03_INIT_CREWMATE` or `PROMPT_04_INIT_IMPOSTOR` directives.
4.  **Sensory Injection:** The test clients will receive their first `PROMPT_06` (Local Map) and `PROMPT_05` (Room State), showing all 9 agents securely locked in the `CAFETERIA`.

*(Note: To kill the background test clients when you are done, use `pkill -f "testclient/main.go"` or close the terminal).*
